Layer 0 Core — Free Tier
Weeks 1–6 ~35 dev days ~$200/mo infra
Next.js 14+ Tailwind CSS Supabase Retell AI Azure Document Intelligence GPT-4o Web Speech API Vercel / Azure pg_cron Web Push
F01 Voice Interface 3d
Web Speech API for capture + Retell AI agent for NLU. Command parser with intent routing: log_dose | schedule_med | query_status | set_reminder. Stream responses for sub-200ms perceived latency.
F02 Med Sheet OCR 6d
Azure Document Intelligence prebuilt model → GPT-4o post-processing to normalize drug names, dosages, frequency. Photo capture UI with crop/rotate. Output: structured JSON → medications table. Critical path — gates F03, F04.
F03 Auto Calendar 3d
Parse OCR output → RFC 5545 iCal events with RRULE. OAuth flows for Google Calendar API + MS Graph. Conflict detection before write. Requires F02.
F04 Reminders 4d
Supabase pg_cron jobs → Web Push (service worker). SMS fallback via Twilio. Escalation chain: push → SMS → family contact. Configurable snooze and ack states in med_logs.
F05 Voice Logging 2d
NLP parse utterances like "I took Metformin at 8am". Named entity recognition for drug + time. Write to med_logs with confidence score. Handle ambiguity prompts.
F06 Adherence Dashboard 3d
React components: daily heatmap, weekly streaks, monthly trend. Color coding: green ≥90%, amber 70–89%, red <70%. Exportable summary for doctor visits.
F07 Family Accounts 4d
Supabase Auth multi-tenant. RBAC: owner | caregiver | viewer. Email invitation flow with token. RLS policies enforce per-user data isolation. family_members junction table.
F08 Document Vault 3d
Supabase Storage with RLS. AES-256 encryption at rest (Supabase default). Categories: labs, prescriptions, EOBs, imaging. Preview (PDF.js) + download. 500MB free tier limit.
F09 Mood Tracker 2d
Daily check-in triggered by morning reminder. 5-point energy/pain/mood scales + symptom checklist. daily_checkins table. Correlate adherence gaps with mood dips.
F10 Doctor Reports 3d
PDF generation (react-pdf). Combine adherence data, mood trends, missed doses. Date range selector. One-tap share via email or QR code in waiting room.
Architecture decision: Supabase Edge Functions for OCR webhook processing keeps Azure Doc Intelligence credentials server-side. Never expose in client bundle.
Architecture decision: pg_cron over application-level cron — survives Vercel cold starts, no separate worker process for reminder delivery.

Layer 1 Intelligence — $9.99/mo
Weeks 7–12 ~40 dev days ~$1,300/mo infra
Mindee API PBM Partner API Stripe Twilio HIPAA Google Calendar freebusy MS Graph freebusy OpenFDA API
F11 Insurance Card OCR 3d
Mindee US Health Insurance pre-built model. Extract: member ID, group number, payer name, copay tiers, Rx BIN/PCN. Store encrypted in insurance_cards. Gates F12, F13.
F12 Benefits Intelligence 6d
Insurance API queries using data from F11. Display: deductible met/remaining, out-of-pocket max, formulary tier per medication, prior auth requirements. Cache with 24h TTL.
F13 Coupon Finder 5d
PBM partner API (GoodRx-style). Query by drug name + zip. Compare insurance copay vs. coupon price. Rank by patient cost. Referral tracking for revenue share.
F14 Cost Comparison 3d
Pharmacy pricing query by zip radius (10/25/50 mi). Price per unit normalization for quantity variants. Map view with price pins. pharmacy_prices table with geo index.
F15 Calendar Conflicts 2d
Google Calendar + MS Graph freebusy API before proposing appointment times. Overlay med schedule conflicts (e.g., fasting bloodwork vs. breakfast pills). Visual conflict flagging.
F16 Family SMS Coordination 6d
Twilio HIPAA messaging (BAA required). Poll family availability via SMS before scheduling. Collect responses, build consensus window, propose to patient. State machine: polling → collecting → resolved.
F17 Appointment Prep 2d
Template system keyed by appointment type (PCP, specialist, lab, imaging). Auto-populate from med list + recent checkins. Generate question list. Pre-fill forms PDF.
F18 Multi-Patient 4d
Account switcher in UI for caregivers managing multiple family members. Per-patient data isolation via RLS. Active patient context in React state + cookie. Separate Stripe subscription per patient.
F19 Med Interactions 3d
OpenFDA drug interaction API on each med_schedule write. Severity classification: major | moderate | minor. Informational display only — no clinical decisions. DISCLAIMER required in UI.
F20 Pre-Auth Tracker 3d
Manual status entry + insurance portal polling where available. Timeline: submitted → payer review → approved/denied → appeal. Push alerts on status changes. coordination_requests table.
Stripe Subscription Integration 3d
Stripe Checkout for tier upgrades. Customer Portal for self-service cancel/downgrade. Webhook handler: customer.subscription.updated | deleted → feature flag toggle in Supabase. Graceful degradation on downgrade (no data loss).
Architecture decision: Feature flags stored in users.subscription_tier column — checked server-side in API routes, not client-side. Prevents bypass via DevTools.
Architecture decision: Twilio HIPAA tier requires signed BAA before SMS sends. Gate F16 behind BAA acceptance checkbox stored in user record.

Layer 2 AI Calls — $29.99/mo
Weeks 13–20 ~55 dev days ~$500 + usage/mo
Retell AI agents Twilio Voice WebSocket audio streaming Call state machine Speaker diarization GPT-4o
F21 Doctor Office Calls 12d
12-state call machine: INIT → DIALING → GREETING → HOLD → IVR_NAV → AGENT_WAIT → AGENT_DETECTED → REQUEST → CONFIRM → COMPLETE → FAILED → LOGGED. IVR navigation via DTMF + speech detection. Human agent detection (energy/latency heuristics). Most complex feature in L2.
F22 Insurance Calls 8d
Benefits verification calls. Claim status inquiries. Prior authorization follow-ups. Member number + provider NPI injection into IVR. Extract structured data (auth number, effective dates) from transcript.
F23 Pharmacy Calls 5d
Prescription status checks. Refill requests by Rx number. Pickup time confirmation. Transfer requests between pharmacies. Simpler IVR trees than F21 — reuse call engine.
F24 Inbound Handling 6d
Provision Twilio number per user. Receive inbound, transcribe, classify (lab result / appointment confirm / prescription ready / general). Summarize + push alert. Route urgent calls to patient's phone.
F25 Call Transcription 4d
Retell AI diarization → speaker labels (Agent / IVR / Patient). GPT-4o summary generation with key facts extracted. Store in call_transcripts. Full-text search via Supabase FTS.
F26 Smart Scheduling 5d
Always runs F16 (family availability poll) BEFORE initiating appointment booking call. Passes confirmed availability window to F21 agent. Prevents booking conflicts automatically.
F27 Lab Alerts 3d
Intercept inbound lab result calls via F24. Extract result values via NLP. Flag abnormal values (pattern match against reference ranges). Alert patient. Optional: forward audio to doctor's office.
F28 Refill Automation 4d
Depletion detection: med_logs + days_supply → estimated empty date. Trigger refill call 7 days prior. Uses F23 pharmacy call engine. Confirmation push notification. Chain: F02 → F04 → F28.
F29 Dispute Assistance 4d
Appeal letter drafting (GPT-4o) from denial reason + clinical notes. Reference relevant CMS/NAIC policy codes. Insurance call agent reads script to request reconsideration. Output: PDF appeal letter + call log.
F30 Second Opinion 4d
Provider search by specialty + zip + insurance. Rank by accepting-new-patients status. F21 call engine to schedule consultation. Transfer records request via F08 vault export.
Architecture decision: Call state machine persisted in call_states table, not in-memory. Allows recovery after Twilio webhook retries and prevents duplicate actions on resume.
Architecture decision: Bill AI call usage (Twilio + Retell) as metered Stripe add-on above flat $29.99. Prevents margin blowout from power users. Transparent cost per call shown in dashboard.

Layer 4 Marketplace — Revenue Share
Month 12–18 ~35 dev days ~$200/mo infra
PBM affiliate API Contextual ad server Affiliate tracking pixels Video SDK (Daily.co / Zoom SDK) ClinicalTrials.gov API
F38 Rx Marketplace 8d
PBM affiliate API. Surface discount cards, mail-order options, specialty pharmacy routing. Click-to-fill referral tracking. Revenue: $2–15 per fill referral depending on drug class.
F39 Pharma Ads 6d
Contextual ad server — triggered by medication context, NOT behavioral tracking. No cross-user profiling. E.g., Metformin user sees Jardiance comparative info. CPM model with pharma advertisers. Must display "Sponsored" label.
F40 Supply Marketplace 5d
Affiliate links for diabetes supplies, pill organizers, BP monitors, CGMs. Surface contextually from medication list. Amazon Associates + specialty health affiliates. 3–8% commission.
F41 Services Directory 5d
Curated health service referrals: care navigation, health coaching, dietitians, pharmacists-for-hire. Per-referral fee model. Quality vetting workflow for listing approval.
F42 Clinical Trials 6d
ClinicalTrials.gov API + pharma direct partnerships. Match patient profile (diagnosis, medications, demographics) against eligibility criteria. Pharma pays referral fee ($50–500) per qualified enrollment lead.
F43 Telehealth 5d
Video visit integration (Daily.co embedded or Zoom SDK). Pre-populate provider with adherence data + mood report from F10. Revenue share 10–15% of visit fee. Warm handoff from AI call scheduling (F21).

Layer 5 Enterprise — Custom Pricing
Month 18–24 ~70 dev days ~$500/mo base
FHIR R4 HL7v2 Epic SMART on FHIR Cerner OAuth Multi-tenant Postgres Theming engine PMPM billing De-identified analytics
F44 White-Label 12d
Multi-tenant architecture: separate Supabase schema per tenant or row-level tenant_id. Theming engine: CSS custom properties overrides via JSON config. Custom domains via Vercel wildcard. License per-member-per-month.
F45 Provider Dashboard 10d
Aggregated patient population view for care teams. Consent management: patient grants provider read access. Non-adherence alerts panel. Risk stratification scoring. HIPAA audit log of all provider access.
F46 Payer Integration 10d
PMPM billing model for payer contracts. Member enrollment via payer data feed. Adherence data reporting to payer for Star Ratings / HEDIS. Outcome-based contract hooks.
F47 Home Health Platform 8d
Agency workflow management: assign patients to aides, visit scheduling. RPM data integration (blood pressure, glucose, weight). Auto-generate CMS OASIS documentation from voice logs.
F48 Pharma Programs 8d
Patient assistance program (PAP) management. Eligibility screening against program criteria. Application routing and status tracking. Co-pay card activation. Pharma SaaS license model.
F49 Population Health 10d
De-identified analytics pipeline (k-anonymity, differential privacy). Adherence cohort analysis. Drug class performance benchmarks. API for pharma research data licensing. Safe harbor certification required.
F50 EHR Integration 12d
FHIR R4 resource mapping: MedicationRequest, Patient, Observation. SMART on FHIR OAuth for Epic/Cerner launch context. HL7v2 ADT feeds for inpatient events. CDS Hooks for real-time clinical decision support injection.
Architecture decision: F44 white-label uses schema-per-tenant for enterprise isolation, not row-level RLS. Row-level acceptable for SMB tiers; dedicated schema required for payer and health system contracts (SOC2/HIPAA audit requirements).
Architecture decision: F50 EHR — prioritize Epic SMART on FHIR first (covers ~35% of US hospitals). Cerner second. Do not attempt HL7v2 custom integrations without a dedicated integration engineer; use a middleware layer (Redox, Mirth Connect) to normalize.
